290 /**
291  * <b>Summary:</b><blockquote>
292  *     An enhanced version of the HTML SELECT tag, populated dynamically<br>
293  * </blockquote>
294  * An enhanced version of the HTML SELECT tag, populated dynamically. It works<br>
295  * very nicely with very large data sets because it can load and page data as needed.<br>
296  * It also resembles ComboBox, but does not allow values outside of the provided ones.<br>
297  * If OPTION tags are used as the data provider via markup, then the<br>
298  * OPTION tag's child text node is used as the displayed value when selected<br>
299  * while the OPTION tag's value attribute is used as the widget value on form submit.<br>
300  * To set the default value when using OPTION tags, specify the selected<br>
301  * attribute on 1 of the child OPTION tags.<br>
302  * <br>
303  * Similar features:<br>
304  * - There is a drop down list of possible values.<br>
305  * - You can only enter a value from the drop down list.  (You can't<br>
306  * enter an arbitrary value.)<br>
307  * - The value submitted with the form is the hidden value (ex: CA),<br>
308  * not the displayed value a.k.a. label (ex: California)<br>
309  * <br>
310  * Enhancements over plain HTML version:<br>
311  * - If you type in some text then it will filter down the list of<br>
312  * possible values in the drop down list.<br>
313  * - List can be specified either as a static list or via a javascript<br>
314  * function (that can get the list from a server)
315  * @constructor
316  * @class
317  * @extends dijit.form.MappedTextBox
318  * @extends dijit.form.ComboBoxMixin
319  * @returns {dijit.form.FilteringSelect} new instance
320 */
